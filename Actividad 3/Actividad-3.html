1. HTML

Introducción

¿Qué es HTML?

HTML significa HyperText Markup Language. Imagínatelo como el esqueleto de una página web. Le dice al navegador qué es cada cosa: esto es un título, esto es un párrafo, esto es una imagen. Sin HTML no hay nada en la pantalla — es la estructura básica de cualquier sitio web que hayas visitado.

Referencia: MDN Web Docs - Primeros pasos con HTML

⬛⬛⬛
Explica las etiquetas head y body.

La etiqueta <head> es como la parte trasera del escenario en un teatro — el público no la ve, pero es importante. Ahí se pone el título de la página, los links al CSS y otras configuraciones.

La etiqueta <body> es todo lo que el usuario sí ve: textos, imágenes, botones, menús, etc. Si está en la pantalla, está dentro del body.

Referencia: MDN Web Docs - ¿Qué hay en el head?

🟥🟩🟦

Etiquetas básicas

Explica las etiquetas: section, article, aside, header, footer.

Estas etiquetas no cambian cómo se ven las cosas por sí solas — solo ayudan a organizar el contenido para que tenga más sentido. Imagínalas como cajas con etiquetas:

<header> — La parte de arriba de la página, normalmente tiene el logo y el menú de navegación.
<footer> — La parte de abajo, normalmente tiene datos de contacto o copyright.
<section> — Un bloque genérico de contenido, como un capítulo de un libro.
<article> — Contenido que tiene sentido por sí solo, como una noticia o un blog post.
<aside> — Contenido relacionado pero no lo principal, como una barra lateral o un dato curioso.

2. CSS

Introducción

¿Qué es CSS?

CSS significa Cascading Style Sheets (Hojas de Estilo en Cascada). Si el HTML es el esqueleto de una página web, el CSS es la ropa y el maquillaje. Controla cómo se ve todo: colores, tamaños, fuentes, espaciado, etc. Sin CSS, todos los sitios web se verían como un documento de Word simple de los años 90.

Referencia: MDN Web Docs - ¿Qué es CSS?

¿Cuál es el propósito del CSS?

El propósito del CSS es separar el contenido (HTML) del diseño (cómo se ve). Esto es útil porque puedes cambiar todo el aspecto de un sitio web editando solo un archivo CSS, sin tocar el HTML para nada. También hace que los sitios se vean bien en diferentes tamaños de pantalla como teléfonos, tablets y computadoras.

Referencia: MDN Web Docs - ¿Qué es CSS?

⬟⬟⬟
¿Cómo se aplica CSS a un HTML? (las 3 opciones)

Opción 1 — Archivo externo: Creas un archivo .css separado y lo enlazas en el <head> así:
<link rel="stylesheet" href="styles.css">
Esta es la forma más común y recomendada.

Opción 2 — Interno (dentro del HTML): Escribes el CSS directamente dentro de una etiqueta <style> en el <head>. Sirve para páginas pequeñas pero mezcla contenido con diseño.

Opción 3 — En línea (dentro de cada etiqueta): Escribes el estilo directo en el elemento HTML así:
<p style="color: red;">Hola</p>
Es la menos recomendada porque se pone muy desordenado rápido.

Referencia: MDN Web Docs - Cómo se estructura el CSS

¿Qué es un selector? ¿Qué es una regla CSS?

Un selector es la forma en que le dices al CSS qué elemento HTML quieres modificar. Por ejemplo, escribir p selecciona todos los párrafos, y escribir .titulo selecciona todo lo que tenga la clase "titulo".

Una regla CSS es la instrucción completa: el selector más lo que quieres cambiar. Por ejemplo:
p { color: blue; font-size: 16px; }
Todo eso junto es una regla. El p es el selector, y todo dentro de las llaves es lo que cambia.

CSS Básico

Explica: Tag Selector, Class Selector, Tag and Class Selector, Id Selector

Tag Selector: Apunta a todos los elementos de ese tipo. Si escribes h1 { color: red; } todos los h1 de la página se ponen rojos, sin excepción.

Class Selector: Apunta solo a los elementos que tienen una clase específica. Se escribe con un punto antes del nombre: .resaltado { background: yellow; }. Varios elementos pueden compartir la misma clase.

Tag and Class Selector: Combina los dos — solo se aplica si el elemento es ese tipo de etiqueta Y además tiene esa clase. Ejemplo: p.resaltado { font-size: 20px; } — esto solo afecta a los párrafos con la clase "resaltado", no a los divs con la misma clase.

Id Selector: Apunta a un elemento único. Se escribe con un hashtag: #menu { width: 200px; }. Cada id solo debe aparecer una vez en la página — es como un nombre propio, no una etiqueta.

Referencia: MDN Web Docs - Selectores CSS

¿Qué es una clase? ¿Qué es un id? ¿Cómo se relacionan con CSS?

Una clase es una etiqueta que le pones a uno o más elementos HTML para poder estilizarlos juntos. Se agrega así: <p class="importante">. Muchos elementos pueden tener la misma clase.

Un id es un nombre único para un elemento específico. Se agrega así: <div id="menu-principal">. Solo un elemento por página debe tener el mismo id.

Se relacionan con CSS porque CSS los usa como selectores. Las clases se seleccionan con punto (.importante) y los ids con hashtag (#menu-principal).

Referencia: MDN Web Docs - Selectores de tipo, clase e ID

Explica el CSS Box Model. Muestra un ejemplo.

Cada elemento HTML es básicamente una caja. El Box Model describe las capas de esa caja de adentro hacia afuera:

Content (contenido) — el texto o imagen de adentro.
Padding — espacio vacío entre el contenido y el borde (adentro de la caja).
Border — la línea que rodea el padding y el contenido.
Margin — espacio vacío afuera del borde, separa este elemento de los demás.

Explica las media queries

Las media queries te permiten aplicar CSS diferente dependiendo del tamaño de la pantalla. Imagina que tu sitio se ve bien en computadora pero horrible en el teléfono — las media queries te dejan arreglar eso diciéndole al navegador: "si la pantalla es más pequeña que X píxeles, usa este CSS en su lugar."

Ejemplo:
@media (max-width: 600px) {
  body { background-color: pink; }
}

Esto significa: si la pantalla mide 600px o menos (como un teléfono), el fondo se pone rosa. En una pantalla grande sigue normal.

Referencia: MDN Web Docs - Media queries

Layout CSS

Explica display: grid

Grid es como poner tu contenido en una tabla con filas y columnas. Tú decides cuántas columnas quieres, qué tan anchas son, y el navegador coloca tus elementos en esas casillas automáticamente.

Ejemplo: si escribes grid-template-columns: 1fr 1fr 1fr obtienes 3 columnas iguales, y tus elementos las van llenando de izquierda a derecha, fila por fila. Es perfecto para layouts de páginas, galerías de fotos, o cualquier vez que necesites cosas alineadas en una cuadrícula.

Referencia: MDN Web Docs - CSS Grid Layout

Explica display: flex

Flex (abreviación de Flexbox) sirve para acomodar elementos en una sola fila o columna. Imagínalo como poner cosas en un estante — puedes decir "ponlos todos en fila, distribúyelos de manera uniforme, y céntralos verticalmente."

La diferencia principal con Grid: Flex trabaja en una dirección a la vez (ya sea fila O columna), mientras que Grid trabaja en ambas direcciones al mismo tiempo. Flex es perfecto para barras de navegación, grupos de botones, o para centrar algo en la pantalla.

Referencia: MDN Web Docs - Flexbox
